# CYP2C9 Pharmacogenomics: Machine Learning Analysis

End-to-end ML pipeline that predicts CYP2C9 metabolizer phenotypes (Poor vs. Intermediate) for diplotypes of indeterminate function, with SHAP-based interpretability and downstream prioritization of variants for MD simulation and clinical review.

## Repository layout

```
.
├── data/                     Raw inputs + intermediate annotation files
├── scripts/
│   ├── 01_annotation/        Annotate alleles/diplotypes, extract features
│   ├── 02_models/            Train RF / XGBoost / SVM models with SHAP
│   ├── 03_downstream/        Model comparison and downstream analysis
│   └── pipelines/            Bash drivers that run the full chain
├── outputs/                  Generated CSVs and tabular results (no figures)
├── requirements.txt
└── README.md
```

## Inputs (`data/`)

**Raw inputs:**
| File | Description |
|---|---|
| `CYP2C9_GENE.xlsx` / `CYP2C9_GENE_Formatted.xlsx` | Diplotypes with curated phenotype assignments (Intermediate, Poor) |
| `CYP2C9_INDETERMINATE_PHENO.xlsx` | Diplotypes of indeterminate function to be predicted |
| `CYP2C9_POOR_METABOLIZER.csv` | Curated list of CPIC poor-metabolizer diplotypes |
| `CYP2C9_allele_definition_table.xlsx` | CPIC allele → variant mapping |
| `variant_annotation.tsv` | PharmGKB variant annotations |
| `clinpgx-summary-annotations.tsv` | ClinPGx allele-level summary annotations |
| `drug_level_annotation.tsv` | Drug-level annotation table |

**Intermediate files** (regenerated by `scripts/01_annotation/`):
`clinical_features_*.csv`, `indeterminate_*_annotations*.csv`, `cyp2c9_variant_scores.csv`, `hybrid_predictions.csv`.

## Scripts

### 01_annotation — input → annotated features
| Script | Reads | Writes |
|---|---|---|
| `extract_clinical_features.py` | `variant_annotation.tsv`, `clinpgx-summary-annotations.tsv`, `CYP2C9_GENE_Formatted.xlsx` | `clinical_features_training.csv`, `clinical_features_indeterminate.csv` |
| `annotate_indeterminate_diplotypes.py` | `CYP2C9_INDETERMINATE_PHENO.xlsx` | `indeterminate_diplotype_annotations.csv` |
| `annotate_indeterminate_alleles.py` | `CYP2C9_GENE_Formatted.xlsx`, PharmGKB annotations | `indeterminate_allele_annotations*.csv`, `indeterminate_region_summary.csv` |
| `annotate_alleles_with_phenotype.py` | `CYP2C9_GENE_Formatted.xlsx`, `CYP2C9_INDETERMINATE_PHENO.xlsx` | `allele_phenotype_annotations.csv` |
| `annotate_with_scores.py` | `CYP2C9_allele_definition_table.xlsx`, `indeterminate_diplotype_annotations.csv` | `cyp2c9_variant_scores.csv`, `indeterminate_diplotype_annotations_with_scores.csv` |

### 02_models — features → predictions + SHAP
Each script trains a model and writes `metrics.csv`, `predictions.csv`, `full_predictions.csv`, `feature_importance.csv`, `confusion_matrix.csv`, plus SHAP plots and figures (figures are not included in this repo but are produced when scripts are re-run).

| Script | Algorithm | Output directory |
|---|---|---|
| `cyp2c9_clean_model.py` | Random Forest (48 features) | `clean_model_output/` |
| `cyp2c9_xgboost_validated.py` | XGBoost (validated — canonical final model) | `xgboost_validated_output/` |
| `cyp2c9_xgboost_model_v2.py` | XGBoost v2 | `xgboost_model_output/` |
| `cyp2c9_svm_model.py` | SVM | `svm_model_output/` |

### 03_downstream — model outputs → comparison and clinical prioritization
| Script | Purpose |
|---|---|
| `compare_models_validated.py` | 3-model (RF/XGBoost/SVM) head-to-head with CV and held-out metrics |
| `create_model_comparison_figure.py` | Figure 3: model performance comparison |
| `downstream_analysis.py` | Pathogenicity scoring, MD simulation candidates, clinical recommendations |

### pipelines — drivers
| Script | Runs |
|---|---|
| `run_full_pipeline.sh` | Random Forest end-to-end |
| `run_full_pipeline_xgboost.sh` | XGBoost end-to-end |
| `run_full_pipeline_XG.sh` | XGBoost (alternate driver) |
| `run_full_pipeline_svm.sh` | SVM end-to-end |

> Note: the pipeline drivers also invoke manuscript-generation scripts that are **not included** in this repository (this repo is scoped to analysis only). Comment out or remove those steps from the `.sh` files when reproducing the analysis.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Reproducing the analysis

```bash
cd scripts/02_models
# Run the canonical (validated XGBoost) model
python3 cyp2c9_xgboost_validated.py
# Or run an alternate model
python3 cyp2c9_clean_model.py
```

Scripts expect input files to be in the working directory; either run from `data/` or symlink the input files next to the script.

## Outputs (`outputs/`)

Pre-computed tabular results from a prior run are included for reference (figures are excluded — regenerate by re-running the scripts). Each model's directory holds its metrics, predictions, feature importance, and confusion matrix. `downstream_analysis_output/` holds the pathogenicity scores, MD simulation candidates, and clinical recommendation tables.
